1. What this framework solvesO que este framework resolveLo que este framework resuelve

AI coding agents in enterprise environments share a common failure pattern. They reconstruct context on every interaction, infer architecture from raw files, and generate code that violates conventions the codebase already encodes. The cost is measurable in three dimensions: tokens consumed, hallucinations introduced, and developer trust eroded.

Agentes de codificação IA em ambientes corporativos compartilham um padrão de falha comum. Reconstroem contexto a cada interação, inferem arquitetura de arquivos brutos, e geram código que viola convenções que a base já codifica. O custo é mensurável em três dimensões: tokens consumidos, alucinações introduzidas, e confiança do dev erodida.

Los agentes de codificación IA en ambientes corporativos comparten un patrón de falla común. Reconstruyen contexto en cada interacción, infieren arquitectura desde archivos crudos, y generan código que viola convenciones que la base ya codifica. El costo es medible en tres dimensiones: tokens consumidos, alucinaciones introducidas, y confianza del dev erosionada.

A 2025 study by METR found experienced developers took 19% longer to complete tasks when using AI tools in large, mature repositories, with fewer than half of AI outputs accepted as-is. The cause was not the model's reasoning capability; it was the absence of structured context about how the codebase actually works.

Four agent integration patterns (source 06)Quatro padrões de integração de agente (fonte 06)Cuatro patrones de integración de agente (fuente 06)

  • Pattern 1: Static File Injection. Recipes write context as files alongside the codebase; agents read them directly.
  • Pattern 2: MCP Server Integration. Context exposed via Model Context Protocol servers (Moderne Moddy is one example), queryable on demand.
  • Pattern 3: Custom Agent Configuration. Context bundled into agent system prompts and skill definitions for vendor-specific tooling.
  • Pattern 4: Hybrid Distribution. Combine the above. Static for stable knowledge, MCP for mutable state, custom for tool-specific needs.

7. Five core principlesCinco princípios centraisCinco principios centrales

Non-negotiable. Apply uniformly across implementations.

Não negociáveis. Aplicam-se uniformemente em todas as implementações.

No negociables. Se aplican uniformemente en todas las implementaciones.

  • Determinism over inference. Context is derived through deterministic analysis, not probabilistic retrieval. Same input always produces same output.
  • Reusability over recomputation. Context is computed once, versioned, reused across agent sessions, tasks, tools.
  • Vendor neutrality over lock-in. The framework specifies interfaces and artifacts, not products. Any compliant implementation can replace any other.
  • Auditability over magic. Every artifact is human-readable, traceable to a recipe execution, signed with a content hash.
  • Refresh over staleness. Context is treated as a build artifact, regenerated on a defined cadence, invalidated when code changes materially.
