1. The sprawlA dispersãoLa dispersión

Modern enterprise development environments suffer from a specific and compounding form of sprawl. APIs proliferate without a central inventory. Shadow APIs get built because teams cannot discover what already exists. VS Code extensions are installed without vetting. AI agents and plugins pull data into external services without oversight. MCP servers are configured ad hoc by individual developers.

Os ambientes corporativos modernos sofrem com uma forma de dispersão específica e composta. APIs proliferam sem inventário central. Shadow APIs nascem porque os times não conseguem descobrir o que já existe. Extensões VS Code são instaladas sem vetting. Agentes de IA e plugins puxam dados para serviços externos sem supervisão. Servidores MCP são configurados ad hoc por cada desenvolvedor.

Los entornos empresariales modernos sufren una forma específica y compuesta de dispersión. Las APIs proliferan sin inventario central. Las shadow APIs nacen porque los equipos no pueden descubrir lo que ya existe. Las extensiones VS Code se instalan sin revisión. Los agentes de IA y plugins envían datos a servicios externos sin supervisión. Los servidores MCP se configuran ad hoc por cada desarrollador.

  • Security risk. Unvetted extensions and MCP servers may exfiltrate code or credentials.
  • Compliance risk. Sensitive data reaches AI model APIs without authorization.
  • Productivity risk. Developers duplicate APIs, use deprecated interfaces, and burn time on tooling setup.
