Fundamentos.

O que é um token, como vira dinheiro e por que agentes mudam a equação por um fator de 40 a 400.

Fundamentos · 01

Token não é palavra. É a unidade de trabalho do LLM.

01BPE tokenization: "GitHub Copilot" = 3 tokens. "copilot" = 1. Texto técnico custa mais que texto comum.
02Três categorias: input (o que você envia), output (o que o modelo gera), cached (o que o modelo já viu antes). Cada uma tem preço distinto.
03Context window: tudo que o modelo "vê" por chamada. Quanto maior, mais caro. Agentes acumulam contexto a cada tool call.
Fundamentos · 02
400×O multiplicador agêntico

Uma sessão de chat custa ~5K tokens. A mesma pergunta em agent mode com 20 tool calls: 200K a 2M tokens. O modelo não mudou. O padrão de uso sim.

Parte II
II

Disciplinas técnicas.

Context engineering, prompt caching, design de agentes e estratégias token-eficientes.

Disciplinas técnicas · 01

Context engineering: o que o modelo vê determina o custo.

01Context rot: cada tool call adiciona tokens que não saem. Sessões longas incham rápido. Solução: compaction a 70% do window.
02AGENTS.md curado: instrução persistente, lida uma vez por sessão. Evita repetição a cada prompt. Curadoria humana supera LLM-gerado.
03Tool result clearing: após processar resultado de tool, remove da história. Reduz 30-50% em sessões longas.
04Subagent fan-out: contexto isolado por subtarefa. Cada subagente retorna resumo de ≤2K tokens. Redução de 70-90%.
Disciplinas técnicas · 02

Prompt caching: leia o mesmo bloco 10×, pague 1×.

Funciona
Prefixo estável no topo
  • Sistema de prompt fixo com cache_control TTL 1h
  • AGENTS.md e regras de projeto: mudança semanal
  • Cache read custa 10% do input fresco
  • Hit rate alvo: >60% em workflows conversacionais
Quebra o cache
Anti-padrões que invalidam
  • Timestamp ou ID aleatório no prompt (invalida sempre)
  • Reordenar blocos entre chamadas (zera o hash)
  • Conteúdo dinâmico antes do estático (prefix errado)
  • Prefixo abaixo de 1.024 tokens (cache não ativa)
Disciplinas técnicas · 03

Workflow ou agente? 70% dos casos: workflow.

01Workflow: sequência determinista. Custo previsível, fácil de auditar, baixa latência. LLM faz partes, código decide o próximo passo.
02Agente: LLM decide o próximo passo. Custo variável (10× entre execuções). Use quando espaço de solução é amplo e varia por input.
03Os 5 padrões Anthropic: Prompt Chaining, Routing, Parallelization, Orchestrator-Workers, Evaluator-Optimizer. Compostos, não frameworks.
04Regra: comece com workflow. Adicione agência só quando workflow demonstravelmente não é suficiente.

A mudança de junho.

GitHub Copilot migra de PRUs para billing por tokens em 1 de junho de 2026. Sete mudanças concretas.

A mudança · 01

7 mudanças concretas a partir de 1 de junho de 2026.

01PRUs aposentados em planos mensais. Substituídos por GitHub AI Credits (1 Credit = US$ 0,01).
02Completions e bundled models (GPT-4.1, GPT-5 mini) continuam incluídos sem consumir créditos.
03Fallback automático removido: créditos zerados bloqueiam modelos premium (bundled continua).
04Code Review agora consome AI Credits e Actions minutes simultaneamente.
05Planos anuais Pro/Pro+: multiplicadores sobem 3,6× a 6×. Capacidade real cai proporcionalmente.

Governança e FinOps.

Inform, Optimize, Operate. Métricas, observabilidade, e Azure AI Foundry no ecossistema.

Governança · 01

FinOps Foundation: três fases iterativas.

01Inform: visibilidade granular. Quem usa, quanto, com qual modelo, em qual feature. Sem dados, otimização é chute.
02Optimize: identificar oportunidades específicas com payback estimado. Routing, cache, tool design, treinamento. Cada ação é mensurável.
03Operate: ciclo contínuo. Reuniões mensais de FinOps, dashboards, ajuste de policy, retreinamento de devs. Cultura, não evento.

Aplicação prática.

8 patterns com código, 10 anti-padrões catalogados e timeline de 16 semanas.

Aplicação prática · 01

8 patterns prontos para implementar amanhã.

Pattern 01
Routing
Classificador dinâmico decide modelo. 40-70% de redução no custo médio.
Pattern 02
Cache 3-tier
Exato + semântico + provider. Hit rate combinado 60-90%.
Pattern 03
Fan-out
Contexto isolado por subtarefa. 70-90% em workflows exploratórios.
Pattern 04
Plan/Exec
Opus planeja, Sonnet executa. 64% mais barato que tudo em Opus.
Aplicação prática · 02

10 anti-padrões de pilotos em América Latina 2025-2026.

✗ 01
Opus como default
Paga $15/M para tarefas que Sonnet ($3) resolve igual
✗ 02
Tudo é agente
Workflow resolveria 70% a 1/5 do custo
✗ 03
System prompt 30K
Dilui sinal, invalida cache, aumenta custo
✗ 04
Sem critério de parada
Loops sem max_iter podem rodar indefinidamente
✗ 05
Cache que não cacheia
Timestamp no prompt invalida sempre
✗ 06
Tool result blowup
read_file sem limite retorna 50K tokens
✗ 07
Contexto longo indevido
1M tokens no Gemini a $10 por chamada
✗ 08
Multi-agent sem coord.
Handoffs consomem 30% dos tokens da sessão
✗ 09
Agent graveyard
80% dos agentes criados não são usados após 3 meses
✗ 10
Sem instrumentação
Custo real chega como surpresa no fim do mês
