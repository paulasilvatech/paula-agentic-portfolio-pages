Workforce demographics. 15% annual COBOL developer loss in Brazil. AI-assisted modernization is not optional, it is workforce-critical. Most enterprises are at organizational maturity Level 0-1 (no agentic AI adoption).

Regulatory forcing functions

Three deadlines that make modernization a compliance requirement, not a strategic option.

01
BACEN 2028. The Brazilian Central Bank requires modern controls, API-based open banking interfaces, and enhanced cybersecurity by 2028. Mainframe platforms cannot deliver these natively.
02
LGPD (Lei Geral de Proteção de Dados). Data governance capabilities (consent, lineage, right to erasure) that mainframe architectures were never designed to support.
03
SOX, ISAE 3402, PCI-DSS. Modern security controls, encryption practices, and audit mechanisms that legacy struggles to provide.

The window: 24-36 months. Agentic AI acceleration is essential to meet these compliance targets without big-bang risk.

II

The framework.

Gartner's 5R, Microsoft's four approaches, and a portfolio approach that lands each application in the right pattern.

Gartner's 5R modernization framework

Five strategies, ranked by transformation depth and risk.

Portfolio approach: no single R fits all workloads. Classify the estate, apply the right pattern per tier. Gartner predicts 75%+ of enterprises will choose incremental over big-bang by 2030.

Microsoft's four modernization approaches

Each maps to a Gartner R, with concrete Azure tooling.

Microsoft approachMaps toToolingWhen
MigrateR02 RehostAzure Migrate, Azure VM, mainframe emulationStable systems, infra cost driver
ModernizeR03 RefactorMainframe Modernization Agent, Azure AI Foundry, Claude on FoundryCOBOL → Java/.NET, AI-accelerated
RearchitectR04 RearchitectAzure Functions, AKS, Service Bus, Event Grid, Cosmos DBStrategic capability, event-driven
ReplaceR05 ReplaceDynamics 365, Power Platform, ISV solutions, Microsoft FabricCommodity processes, vendor solutions exist

Critical insight on GenAI scope (Gartner G00822567). Current GenAI tools excel at code translation but explicitly exclude architecture modernization from their scope. Simple translation creates "JOBOL" - COBOL logic implemented in Java, which represents insufficient modernization. Effective GenAI-assisted modernization requires combining LLMs with domain tools and rule-based systems.

The decision matrix

Where each application lands depends on two axes: business criticality and technical complexity.

High business criticality
Refactor or Rearchitect with AI
  • R03 Refactor if architecture is sound: AI-driven COBOL → modern language
  • R04 Rearchitect if architecture is the bottleneck: redesign boundaries first, then translate
  • AI accelerates discovery and translation; humans approve target architecture
  • Coexistence period 12-24 months mandatory
Lower business criticality
Retain, Rehost, or Replace
  • R01 Retain if system is stable and end-of-life is near
  • R02 Rehost if cloud cost optimization is the only goal
  • R05 Replace if commodity SaaS exists (HR, finance, CRM)
  • Reserve AI investment for the high-criticality estate

Anti-pattern: applying R03 Refactor uniformly across the estate. The goal is not "modernize everything", it is "land each application in the right pattern".

Part III · ~10 min
III

The AI advantage.

Four capabilities that flip the cost equation, three things AI does NOT replace, and the 70/30 operating model.

Four AI capabilities that change the game

Each replaces a previously manual, expensive bottleneck.

CAPABILITY 01
Automated discovery
Claude Code maps dependencies across thousands of files, identifies entry points, traces execution paths, surfaces couplings even experienced developers may miss.
CAPABILITY 02
Risk and opportunity mapping
AI scores each component on complexity, coupling, test coverage, business criticality. Replaces subjective expert judgment with data-driven assessment.
CAPABILITY 03
Strategic planning + oversight
AI generates target architectures, migration sequences, resource estimates. Human architects review and approve. Strategic decisions stay under expert control.
CAPABILITY 04
Incremental implementation
AI translates one module at a time, with validation at every step. Not a black-box conversion. Every output is reviewable and reversible.

Cost risk warning (Gartner G00827372, "When to Use or Not to Use AI Agents", June 2025): agent costs on pay-per-use can scale rapidly with unknown benefits, potentially producing negative business cases if unbounded. Iterative validation gates and human oversight are mandatory.

What AI does NOT replace

Six decisions where human judgment is non-negotiable.

01
Final approval of target architecture decisions. AI proposes; humans choose.
02
Validation of business rule preservation. Subject matter experts certify the new code reproduces the legacy behavior.
03
Performance tuning for specific workload patterns. Production telemetry beats AI projection.
04
Production cutover decisions. A human signs the go-live, accepts the rollback plan.
05
Regulatory compliance sign-off. The accountable executive owns the audit trail.
06
Escalation handling for edge cases. When AI cannot resolve, route to a domain expert.

Target operating model: 70-80% AI-executed, 20-30% human oversight. Consistent with Gartner's AI-assisted productivity estimates. IDC US54300026: only 45% of AI projects deliver measurable business value, ROI framework and governance gates are the difference.

Claude multi-model orchestration

Three models. One pipeline. Cost-optimized by design.

CLAUDE OPUS 4.6
Orchestrator + deep analysis
Primary reasoning engine. Receives high-level tasks, decomposes them, assigns to workers, consolidates results. Uses full 1M token context. Architectural recommendations.
CLAUDE SONNET 4.6
Worker · translate, test, document
Executes specific tasks: code translation, test generation, documentation, PR creation. Focused scope, clear input/output contract. Cost-balanced.

Syntax validation, pattern matching, triage classification, CI/CD checks. Optimized for speed and cost. The first pass on every artifact.

Routing rule: Haiku scans, Sonnet translates, Opus analyzes. The Foundry Model Router optimizes cost automatically. Validated by Gartner research [G00822558].

Three orchestration patterns

Pick the pattern by task shape, not by team preference.

PATTERN 01
Sequential
Step-by-step with handoff. Discovery → Analysis → Translation → Tests → PR. Each step's output feeds the next. Use when stages are well-defined and dependencies linear.
PATTERN 02
Handoff
Agent-to-agent delegation. Translation agent escalates ambiguity to Analysis agent, returns. Use when an agent meets a decision it cannot resolve.
PATTERN 03
Group chat
Collaborative reasoning. Multiple agents debate target architecture options. Orchestrator consolidates. Use for high-stakes design decisions.

Microsoft Semantic Kernel implements all three patterns natively. Dual orchestration (Temporal for durability + LangGraph for branching) is the production pattern for long-running modernization workflows.

Part V · ~14 min
V

The seven-phase journey.

From assessment to factory model. Decision gates at every phase. Reversibility at every step.

The seven-phase journey

Linear when it works. Iterative when it does not.

Phase 1 + 2 · Weeks 1-4

Assessment and carving. Slicing the monolith into modernizable units.

Phase 1: Assessment
Code archaeology with Discovery agent
  • Scan entire codebase: programs, copybooks, JCL, data structures
  • Build dependency graph, trace CALL relationships
  • Map JCL job flows, catalog business rules
  • Output: JSON inventory + Markdown report + Mermaid diagrams
  • Decision gate: portfolio classified into 5R buckets
Phase 2: Carving
Tiered dependency difficulty model
  • Tier 1, leaf modules: low coupling, easy to carve first
  • Tier 2, intermediate: some shared state, moderate difficulty
  • Tier 3, core: high coupling, save for last
  • SMF runtime behavior analysis informs the slicing order
  • Decision gate: approved modernization sequence
Phase 3 + 4 · Weeks 3-8 · Ongoing

Translation and performance assurance. Match mainframe speed before cutover.

Phase 3: Translation
Module-by-module, never big bang
  • Sonnet 4.6 translates COBOL → Java/.NET/Python idiomatically
  • Preserve business semantics, NOT syntactic structure (avoid JOBOL)
  • Generate behavioral equivalence tests in parallel
  • Each module → PR with mapping documentation
  • Decision gate: human review of every PR
Phase 4: Performance
Match mainframe speed parity
  • SMF data establishes baseline (CPU, I/O, response times)
  • Load tests with production-like volume on the modernized service
  • Profile hot paths, optimize critical sections
  • Benchmark vs mainframe, no regression beyond defined SLOs
  • Decision gate: performance sign-off by SRE team
Phase 5 · Coexistence · 12-24 months

The strangler fig pattern. Mainframe and modern systems run in parallel during transition.

Phase 6 · Testing · Continuous

Shadow traffic and behavioral equivalence. Prove parity before flipping the switch.

01
Shadow traffic. Production requests are duplicated to the modern service in read-only mode. Responses compared, not used. Identifies divergences before they hit users.
02
Behavioral equivalence tests. Generated by Test agent during translation. Validate that the modernized code produces identical outputs to the original COBOL across thousands of input fixtures.
03
Mutation testing. Inject defects into the modernized code, confirm tests fail. Validates test suite quality, not just code correctness.
04
Edge case mining. Discovery agent reads decades of audit logs, surfaces rare scenarios. Generated test fixtures cover them explicitly.

Coverage target: 85%+ minimum, with 100% on critical business rule paths. Gartner: AgentOps maturity correlates strongly with successful production cutovers.

Phase 7 · Production cutover + factory model

The human gate. After this, the next 100 modules are factory work.

Cutover decision gate
What must be true before flipping
  • Shadow traffic divergence rate below SLO threshold
  • Performance parity confirmed by SRE team
  • Rollback procedure tested in production-like environment
  • Incident response runbook in place
  • Compliance sign-off from accountable executive
Factory model
After Module 1, the next 100 are repeatable
  • Reusable carving rules, agent prompts, test fixtures
  • Standardized PR template, review checklist
  • Pre-built CI/CD pipeline per target language
  • Cost predictability: token spend per LOC measured
  • Throughput: 50K-500K LOC per quarter sustained
Common pitfalls and mitigations

Six failure modes I have seen in the field, and how to avoid each.

PitfallMitigation
JOBOL. COBOL syntax translated to Java without architectural rethinking.Run a rearchitect pass before translation. AI proposes target boundaries; humans approve.
Big bang cutover. Single switch from legacy to modern.Strangler fig with CDC. 12-24 month coexistence period. Gradual traffic migration.
Test rot. Tests pass but code is wrong because tests inherit the same bugs as the translation.Generate tests from original COBOL behavior, NOT from the translated code. Mutation test the suite.
Agent anarchy. Many agents, no orchestrator, no governance.Single Orchestrator (Opus). Every agent action logged. Approval gates at decision points.
Cost runaway. Token spend exceeds budget before value is delivered.Per-module cost ceiling. Wave 1 → Wave 2 → Wave 3 with cost gates between.
Vendor lock-in. The modernized code only runs on one cloud or with one model provider.Standard, idiomatic target language. No proprietary SDK injection. Multi-model routing.
Governance framework

Auditability is not a feature. It is the difference between a pilot and production.

01
Every agent action logged with structured metadata. Azure Monitor captures who, when, what, why, what changed.
02
Approval gates at decision points. Architecture decisions, production deployments, schema changes pause for human sign-off.
03
RBAC at every layer. Entra ID controls which agents can read which resources. Key Vault holds all secrets.
04
Cost ceilings per workflow. Token spend capped per agent invocation. Runaways trigger circuit breakers.
05
Compliance evidence generated automatically. SOX, ISAE 3402, LGPD, PCI-DSS audit trails as a side effect of normal operation.

Gartner G00823006 · Defend vs Extend

The ROI math: defending costs you 52%, extending earns you 134%.

Gartner research G00823006 quantifies the difference between two postures toward a legacy estate. Defending (status quo, patches only) destroys 52% of value over the period analyzed. Extending (modernization with measured return) creates 134%. The choice is rarely about cost; it is about which curve you accept.

DEFEND
−52%
Status quo, patches only, no modernization. Maintenance debt compounds. Talent attrition accelerates. JOBOL risk creeps in.
EXTEND
+134%
Modernization with measured return. Strangler-fig carving. Performance parity. Coexistence. Decommission only when the new system is provably better.

Gartner G00822567 also flags JOBOL (COBOL logic ported to Java without idiom conversion) as a critical anti-pattern. Avoid it.
